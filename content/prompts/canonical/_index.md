# Canonical prompts — index

The promoted-winners shelf. Prompts here have shipped, performed, and have a metric proving it. Use these first when generating for a similar scope.

---

## Promotion rule

Move a prompt here when ALL of the following hold:

1. **Shipped** — used in a real, live client campaign.
2. **Performed** — beat a defined metric (ROAS, CTR, CPL, CVR, completion rate, lift vs control).
3. **Attribution recorded** — client, campaign, period, and metric written into the prompt file.

Promotion is a curation act, not automatic. A high-performing prompt in a noisy test doesn't qualify; a clean control-tested win does.

---

## File format

Every canonical prompt opens with:

```yaml
---
promoted_from: prompts/<original-path>.md
client: <slug>
campaign: <name>
period: YYYY-MM to YYYY-MM
metric: <e.g. CTR 3.4% vs 1.1% control>
last_reviewed: YYYY-MM-DD
---
```

Then the prompt body, followed by a "context" section explaining the campaign and why this prompt won.
